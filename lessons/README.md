# Lessons

This folder contains lesson-level material that expands the syllabus files into teachable units.

Each lesson follows the quality bar in [Lesson Standards](../LESSON-STANDARDS.md). Lessons should create concrete artifacts and should be grounded in durable engineering, security, reliability, privacy, governance, repeatable delivery, and distributed-systems practices.

## Planned Expansion Order

After Syllabus 01 through Syllabus 16, build lessons in this order:

1. Syllabus 17 - AI Security Architecture
2. Syllabus 18 - Advanced Evaluation and Release Gates
3. Syllabus 21 - Multi-Agent Systems and Orchestration
4. Syllabus 26 - Capstone Sequence

This order keeps architecture, model judgment, client data safety, repo organization, Python implementation depth, API productization, retrieval data engineering, production operating discipline, deployment practice, reliability discipline, repeatable infrastructure delivery, and distributed-systems control together first, then moves into security, orchestration, and capstone proof.
