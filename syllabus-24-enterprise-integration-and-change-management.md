# Syllabus 24: Enterprise Integration and Change Management
**Status:** Enterprise Gap - Making AI Systems Work Inside Real Organizations
**Format:** One chat session per unit, self-paced

---

## What This Is

The best AI system can still fail if it does not fit the organization that receives it. This syllabus teaches you how to integrate with existing tools, uncover real requirements, manage rollout, train users, and hand off ownership so the system survives after delivery.

---

## Unit 1 - Stakeholder Discovery and Requirements

**Goal:** Learn what the organization actually needs before designing the system.

**Topics:**
- Identifying users, buyers, operators, approvers, and blockers
- Functional vs. non-functional requirements
- Current workflow mapping
- Pain points, constraints, and success metrics
- Turning conversations into a clear requirements brief

**Session starter:**
> "Help me run stakeholder discovery for an AI system project. I need to identify users, buyers, operators, approvers, blockers, requirements, constraints, current workflows, and success metrics."

---

## Unit 2 - Integrating with Existing Tools

**Goal:** Connect AI systems to the tools clients already use instead of forcing a new workflow.

**Topics:**
- CRM, Slack, email, Google Workspace, Notion, Airtable, and internal APIs
- Webhooks, polling, scheduled syncs, and direct API calls
- Data mapping between systems
- Permissions and audit trails for integrations
- Designing integrations that fail visibly and recover cleanly

**Session starter:**
> "Teach me how to integrate an AI workflow with existing client tools like Slack, email, Google Workspace, a CRM, or an internal API. Explain webhooks, polling, scheduled syncs, data mapping, permissions, and failure handling."

---

## Unit 3 - Workflow Redesign and Adoption

**Goal:** Make the AI system fit real human work instead of adding another thing to manage.

**Topics:**
- Mapping before and after workflows
- Removing steps instead of adding more screens
- Designing for skeptical users and uneven adoption
- Pilot groups and phased rollout
- Measuring adoption without spying on people

**Session starter:**
> "Help me redesign a human workflow around an AI system. I want to map the current workflow, design the future workflow, reduce friction, plan a pilot, and measure adoption without creating surveillance vibes."

---

## Unit 4 - Training, Documentation, and Support

**Goal:** Prepare users and operators to use the system without depending on you for every question.

**Topics:**
- Quickstart guides for users
- Operator runbooks for admins
- Support paths: bug, feature request, user confusion, and incident
- Training sessions that focus on real tasks
- Writing documentation that survives turnover

**Session starter:**
> "Help me create training, documentation, and support materials for an AI system. I need a user quickstart, an operator runbook, a support process, and a training outline based on real tasks."

---

## Unit 5 - Handoff and Long-Term Ownership

**Goal:** Transfer the system responsibly so it can keep running after your engagement ends.

**Topics:**
- Ownership map: who owns product, operations, security, data, and billing
- Admin access and credential transfer
- Maintenance schedule and review cadence
- Decommissioning plan if the system is no longer useful
- Final handoff package for clients

**Session starter:**
> "Help me design a client handoff package for an AI system. Include ownership, credentials, billing, operations, security, data, maintenance cadence, support process, and decommissioning plan."

---

## Practice Project

Choose one client-style AI system. Write a requirements brief, an integration map, a before-and-after workflow, a rollout plan, a user quickstart, an operator runbook, and a final handoff checklist.
